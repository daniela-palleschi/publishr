Tables from scratch with tribble()

Not all manuscript tables come from data. Methods sections, model specification tables, and condition overviews are often written out manually. tibble::tribble() lets you create a table row by row in a readable format, which can then be piped into gt or kableExtra for formatting.

tbl_conditions <-
  tibble::tribble(
  ~Condition, ~Tense,             ~Lifetime, ~Example,
  "A",        "Present Perfect",  "Living",  "Der Dirigent hat den Komponisten gekannt, der...",
  "B",        "Present Perfect",  "Dead",    "Der Dirigent hat den Komponisten gekannt, der...",
  "C",        "Simple Past",      "Living",  "Der Dirigent kannte den Komponisten, der...",
  "D",        "Simple Past",      "Dead",    "Der Dirigent kannte den Komponisten, der..."
)

if (knitr::is_latex_output()) {
  tbl_conditions |>
    kbl(
      format    = "latex",
      booktabs  = TRUE,
      col.names = c("Condition", "Tense", "Lifetime", "Example stimulus")
    ) |>
    kable_styling(latex_options = c("hold_position", "scale_down")) |>
    add_header_above(c(" " = 1, "Condition factors" = 2, " " = 1)) |>
    column_spec(4, width = "6cm") |>
    kableExtra::footnote(
      general           = "Only the critical verb region is shown.",
      general_title     = "Note.",
      footnote_as_chunk = TRUE
    )
} else {
  tbl_conditions |>
    gt() |>
    tab_style(
      style     = cell_text(weight = "bold"),
      locations = cells_column_labels()
    ) |>
    tab_spanner(
      label   = "Condition factors",
      columns = c(Tense, Lifetime)
    ) |>
    cols_label(Example = "Example stimulus") |>
    cols_width(Example ~ px(300)) |>
    tab_source_note("Only the critical verb region is shown.")
}

Cross-referencing tables

To cross-reference a table in prose, add a label starting with tbl- and a tbl-cap to the chunk options:

```{r}
#| label: tbl-desc
#| tbl-cap: "Total reading time (ms) by condition."
tbl_desc |> gt()
```

Then reference in prose with @tbl-desc:

Descriptive statistics are presented in @tbl-desc.

This renders as “Descriptive statistics are presented in Table 1.” and updates automatically if tables are added or reordered.

Warning

Cross-references only work when the chunk label begins with tbl-. A label like #| label: desc-table will not be recognised as a table reference.

Summary tables

Summary tables in manuscripts serve two purposes: describing the sample (demographics, condition counts) and reporting descriptive statistics (means, SDs, reading times). The appropriate package depends on your target output format.

NoteSimple Markdown tables

For quick, small tables that do not need formatting, Markdown syntax is the simplest option:

| Condition | Tense            | Lifetime |
|---|---|---|
| A         | Present Perfect  | Living   |
| B         | Present Perfect  | Dead     |

Markdown tables render in all output formats without any packages. However they offer no control over column widths, cell formatting, footnotes, or spanning headers. For anything that will appear in a manuscript, gt or kableExtra is preferable.

tbl_desc <- df_clean |>
  filter(region == "verb") |>
  mutate(lifetime = factor(lifetime, levels = c("living", "dead"))) |>
  summarise(
    n      = n(),
    mean   = mean(tt,   na.rm = TRUE),
    sd     = sd(tt,     na.rm = TRUE),
    median = median(tt, na.rm = TRUE),
    iqr    = IQR(tt,    na.rm = TRUE),
    .by    = c(tense, lifetime)
  ) |>
  arrange(tense, lifetime)

Exporting tables

# gt to Word or PDF
tbl_gt <- tbl_desc |> gt()
gtsave(tbl_gt, here::here("output", "tables", "tab_desc.docx"))
gtsave(tbl_gt, here::here("output", "tables", "tab_desc.pdf"))

# kableExtra to LaTeX
tbl_desc |>
  kbl(booktabs = TRUE, digits = 1) |>
  kable_styling() |>
  save_kable(here::here("output", "tables", "tab_desc.tex"))
