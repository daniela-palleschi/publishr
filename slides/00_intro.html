Writing and collaborating

Common tools for manuscript writing

  • Microsoft Word — ubiquitous, familiar, easy to share
  • Google Docs — real-time collaboration, version history
  • Overleaf / LaTeX — precise formatting, journal templates, reference management
  • Quarto / R Markdown — dynamic documents, code and prose together

Note

Most researchers use a combination — and most have a story about something going wrong.

Word and Google Docs

Pros

  • Familiar and widely used
  • Easy to share with co-authors and reviewers
  • Track changes and comments for collaboration
  • No technical setup required

Cons

  • No connection to your analysis
  • Copy-paste errors when updating results
  • Version control is manual (manuscript_final_v3_ACTUAL.docx)
  • Formatting can break between platforms

Overleaf / LaTeX

Pros

  • Precise, consistent formatting
  • Journal templates available
  • Native support for equations and cross-references
  • Bibliography management with .bib

Cons

  • Steep learning curve
  • Still requires manual copy-paste of results
  • Collaboration can be slow without a paid plan
  • Error messages are cryptic

Quarto

Pros

  • Code, prose, and output in one document
  • Results update automatically when data changes
  • Renders to HTML, Word, and PDF from the same source
  • Cross-references, equations, and citations built in

Cons

  • Requires learning Markdown and R
  • Collaborators may not use it
  • PDF output requires LaTeX
  • Debugging render errors can be frustrating

Reporting results

The copy-paste problem

A typical results workflow without Quarto:

  1. Run model in R
  2. Read off coefficients from the console
  3. Type numbers into Word
  4. Reviewer asks to rerun with different exclusion criteria
  5. Rerun model, find updated values
  6. Manually update every number in the manuscript
  7. Hope nothing was missed

Warning

Around 50% of psychology papers contain at least one statistical reporting error, many due to manual transcription (Nuijten et al., 2016).

The Quarto alternative

  1. Run model in R
  2. Save model as .rds
  3. Load in .qmd and report inline: `r report_t(m1, "condition1")`
  4. Reviewer asks to rerun
  5. Rerun model, save updated .rds
  6. Re-render document
  7. Every number updates automatically

This workshop

Workshop structure

Day 1

  • Writing in Quarto: Markdown basics, citations, cross-references
  • Output formats: HTML, Word, PDF, and LaTeX
  • Incorporating data: code chunks, inline reporting

Day 2

  • Open and reproducible research: FAIR, data life cycle
  • Tables: gt, kableExtra, modular workflow
  • Model results: broom, modelsummary, inline reporting
  • Figures: ggplot2, exporting, cross-referencing

Tips on code chunk options woven throughout both days.

How this workshop works

  • Mix of short overviews and hands-on exercises
  • Follow along in your own RProject
  • Red slides = time to pause, discuss, or try something
  • Ask questions any time

The dataset

The exercises use a real dataset based on Palleschi et al. (2025) (osf.io/6sra7), adapted for workshop purposes:

  • Some variables removed, some new ones added
  • Restructured to resemble raw data from a collection platform
  • Participant codes are randomly generated — not real identifiers
  • In Session 2 we will work through anonymisation and data management

What we assume

  • You have used R before
  • You are comfortable with basic tidyverse
  • You have R, RStudio or Positron, and Quarto installed
  • You do not need to be an expert

Learning objectives

By the end of this workshop you will be able to:

  • Write a results section in Quarto that updates automatically
  • Produce output in Word, PDF, and HTML from the same source
  • Extract and report model parameters using broom::tidy()
  • Embed results in prose using inline R code
  • Create publication-ready summary tables and figures

Getting started

Option A: Clone or download the repository

With Git — terminal:

git clone https://github.com/daniela-palleschi/publishr-exercises.git

With Git — RStudio: File → New Project → Version Control → Git

Without Git:

  1. Go to github.com/daniela-palleschi/publishr-exercises
  2. Click Code → Download ZIP
  3. Unzip and open publishr-exercises.Rproj

Option B: Set up your own RProject

  1. File → New Project → New Directory → New Project
  2. In the terminal:
