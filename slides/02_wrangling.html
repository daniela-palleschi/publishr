Data location and protection

Always load from data/raw/ and save processed outputs to data/processed/. Alternatively, have an input/ and output/ folder: you never save anything from your scripts in input/.

If raw data contains personal/identifiable information, never share it publicly (and only ever with collaborators). Once you have done your initial pre-processing and (pseudo-)anonymisation, archive it.

Tip

Use |> (native pipe, R ≥ 4.1) rather than %>% for new code. It requires no package and is slightly faster.

Saving processed data

  • save as csv (write_csv()) or RDS (write_rds()).
write_csv(df_clean, here::here("data", "processed", "exp1_clean.csv"))

Load it back in subsequent scripts:

df_clean <- read_csv(here::here("data", "processed", "exp1_clean.csv"))
